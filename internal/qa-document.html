交流 Camp 网页 QA 测试文档

内部员工测试版 | 测试对象：客户自助问答网页 + 三份 camp brochure | 更新日期：2026-06-29

测试目的：确保客户能通过网页先自行了解三个 camp 产品，并在无法自助解决时顺利转人工邮件到 xletravels@gmail.com。

1. 测试范围

模块 测试内容 通过标准
首页展示 页面标题、三个项目卡片、导航链接、移动端布局 信息清晰，无文字重叠，手机和桌面都可阅读
Brochure 下载 三份 PDF 链接是否打开正确文件 每个链接都能打开对应 brochure
问答功能 输入问题、点击快捷问题、答案匹配 常见问题返回相关答案，未知问题引导转人工
转人工 邮件按钮和 mailto 链接 点击后收件人为 xletravels@gmail.com，主题/正文可带入问题
发布访问 GitHub Pages 访问、资源路径、PDF 资源路径 互联网打开页面后 CSS、JS、PDF 均加载正常

2. 关键测试用例

ID 场景 步骤 预期结果
QA-001 客户选择项目 在问题框输入 “Which camp should I choose?” 并提交 回答解释 AI Robotics、Cultural Exchange、Wellness Retreat 的适合人群
QA-002 青少年科技营咨询 输入 “My child likes robotics and AI” 回答指向 Youth AI & Robotics Camp，并提醒确认日期、价格、要求
QA-003 身心疗愈营咨询 点击或输入 wellness retreat 相关问题 回答推荐 Mind-Body Wellness Cultural Immersion Retreat
QA-004 费用包含项 点击 Inclusions 快捷按钮 回答列出住宿、餐饮、交通、材料、门票、保险、接送等需确认项
QA-005 证件材料 点击 Documents 快捷按钮 回答提醒护照、申请信息、紧急联系人、健康信息、签证材料等
QA-006 未知问题转人工 输入一个网页知识库没有覆盖的问题，例如 “Can I pay by invoice from my school?” 回答建议发邮件到 xletravels@gmail.com
QA-007 邮件按钮 点击 Email human support 默认邮件客户端打开，收件人为 xletravels@gmail.com，正文包含用户问题
QA-008 PDF 链接 依次点击三个 brochure 链接 文件名和打开内容一一对应

3. 内容验收清单

  • 网页没有承诺未确认的价格、固定日期、签证结果或不可控交付。
  • 客户能看到三个项目的定位差异。
  • 客户能下载或打开三份 brochure。
  • 所有人工支持入口都指向 xletravels@gmail.com。
  • 未知或复杂问题不会被网页强行回答，而是建议人工确认。

4. 浏览器与设备测试

设备 浏览器 重点
Windows 桌面 Chrome / Edge 导航、PDF 打开、邮件链接
iPhone Safari 移动端排版、按钮大小、PDF 预览
Android Chrome 问答输入、邮件客户端跳转

5. 发布后回归

  1. 打开 GitHub Pages 公网链接。
  2. 刷新页面确认 CSS 和 JS 加载正常。
  3. 点击三个 brochure 链接。
  4. 提交至少 5 个常见问题和 2 个未知问题。
  5. 点击转人工邮件按钮，确认收件人无误。
